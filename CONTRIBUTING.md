# Contributing to AegisPay

First off, thank you for considering contributing to AegisPay! It's people like you that make AegisPay such a great tool for privacy-preserving crypto payroll.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct. Please be respectful to other contributors.

## How Can I Contribute?

### Reporting Bugs
This section guides you through submitting a bug report for AegisPay.
- **Ensure the bug was not already reported** by searching on GitHub under Issues.
- **If you find a Closed issue that seems like it is the same thing that you're experiencing**, open a new issue and include a link to the original issue in the body of your new one.
- **Include steps to reproduce the bug**, along with your environment details (browser, OS, extension versions).

*Note: If you find a Security Vulnerability, please see our [SECURITY.md](SECURITY.md) and do NOT open a public issue.*

### Suggesting Enhancements
- Open an issue detailing your suggested enhancement.
- Describe the current behavior and the behavior you expect to see.
- Explain why this enhancement would be useful to most AegisPay users.

### Pull Requests
1. Fork the repo and create your branch from `master`.
2. If you've added code that should be tested, add tests.
3. If you've changed APIs, update the documentation.
4. Ensure the test suite passes (`npm test` and `cargo test`).
5. Make sure your code lints.
6. Issue that pull request!

## Local Development Setup

Refer to the Prerequisites & Local Installation section in the [README.md](README.md) for instructions on setting up your local environment for the frontend and smart contracts.
